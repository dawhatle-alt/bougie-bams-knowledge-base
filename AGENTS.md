# Agent Instructions

This repository is the Bougie Bams Markdown knowledge base for human documentation and future vector-store ingestion.

## Documentation Rules

- Preserve Markdown frontmatter.
- Do not remove metadata fields.
- Keep one topic per file.
- Prefer short, retrieval-friendly sections.
- Do not include copyrighted NMJL card hands or official rulebook language.
- Do not closely paraphrase official NMJL rules or annual card content.
- Add related links when creating new articles.
- Keep examples original, generic, and teaching-focused.
- When official interpretation is required, tell readers to consult the current official NMJL card or rulebook.

## Review Priorities

Treat these as high-priority issues:

- Broken or missing metadata.
- Invalid domain, category, or subcategory values.
- Broken related-document references.
- Copyright risks involving NMJL card hands, official rulebook language, or annual card content.
- Long, unfocused sections that will chunk poorly for retrieval.

Codex should review documentation changes for structural consistency, metadata completeness, copyright safety, and vector-store readiness.
